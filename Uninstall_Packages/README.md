🔥 ADB 패키지 삭제 프로그램 (UI 버전) 사용법 총정리

루팅 없이도 기본앱, 통신사앱, 사전탑재 앱까지 패키지명 기반으로 일괄 삭제가 가능한 ADB 패키지 삭제 프로그램(UI 버전)입니다.
UI 기반이라 명령어 몰라도 손쉽게 사용할 수 있으며, 패키지명 관리도 자동으로 txt에 저장됩니다.

⸻

✅ 주요 기능 소개

1. 디버깅 연결 확인
   • 프로그램 실행 → “디버깅 연결 확인” 클릭
   • 현재 기기가 ADB로 정상 연결되었는지 즉시 확인 가능
   • device / unauthorized / offline 상태를 표시해줘서 문제 진단도 쉬움

⸻

2. 현재 앱 패키지명 확인
   • 휴대폰 화면 최상단에 떠 있는 앱의 패키지명을 버튼 한 번으로 확인
   • 별도 명령어 입력 없이 자동으로 UI 화면에 표시됨

⸻

3. 패키지명 추가 → txt 자동 저장
   • “패키지명 추가” 클릭 시 입력한 패키지명을
   패키지목록.txt 파일에 자동 저장
   • 프로그램 종료 후에도 목록이 유지되기 때문에 지속적으로 관리 가능

⸻

4. 패키지 txt 전체 삭제 기능
   • txt에 저장된 모든 패키지명을
   “전체 삭제” 버튼 한 번으로 일괄 삭제
   • 기본앱, 통신사앱 등 삭제가 어려운 앱도 ADB 방식으로 제거 가능
   • install-existing 기반이기 때문에 복구도 가능

⸻

🧭 사용 방법 (단계별)

1️⃣ USB 디버깅 활성화 → PC와 연결
(스마트폰에서 “이 컴퓨터 허용” 체크 必)

2️⃣ 프로그램 실행

3️⃣ 디버깅 연결 확인 클릭
→ ADB 연결 상태 점검

4️⃣ 현재 앱 패키지명 확인
→ 상단 앱 패키지 자동 인식

5️⃣ 패키지명 추가
→ txt에 자동 저장됨

6️⃣ 패키지 txt 전체 삭제 클릭
→ 저장된 모든 패키지가 자동으로 삭제됨

⸻

💡 활용 예시
• 통신사/불필요 사전탑재 앱 정리
• 광고성 앱·중복 앱 제거
• 부모님/지인 스마트폰 가볍게 정리
• 매장 데모폰, 업무용 폰 세팅 최적화
• 루팅 없이 깔끔한 미니멀 환경 만들기

⸻

🛠️ 문제 해결
• unauthorized → USB 디버깅 허용 팝업 확인
• offline → 케이블 교체 / MTP(파일전송) 전환
• 삭제 후 앱 재생성 → 자동복원/동기화 기능 OFF
• 오작동 → install-existing으로 복구 가능
• 삭제 불가 표시 → 관리자 권한으로 실행

⸻

⚠️ 주의사항
• 시스템 핵심 패키지를 삭제하면 기능 제한 또는 부팅 오류 가능
• 반드시 필요한 패키지는 삭제하지 않도록 주의
• 사용으로 발생하는 문제는 본인 책임
